🌐 Protokoll-Labor: Infrastruktur-Grundlagen

Erarbeitung der essenziellen Netzwerkdienste im Selbststudium zur Vorbereitung auf die Ausbildung im Bereich Systemintegration.

1. DHCP (Dynamic Host Configuration Protocol)

Automatisierte Zuweisung von Netzwerkkonfigurationen an Clients im IP-Kreis.

  • Zuweisung von: IP-Adresse, Subnetzmaske, Standard-Gateway und DNS-Servern.
  • Praxis-Architektur: Realisierbar über Standalone-Server (z. B. Linux-Dienst) oder standardmäßig integriert in Routing-Endgeräten im Heimnetzwerk.

2. DNS (Domain Name System)

Hierarchische Namensauflösung im WAN/LAN zur Übersetzung von Domainnamen in maschinenlesbare IP-Adressen.

  • A-Record: Auflösung in klassische IPv4-Adressen (beschränkter Adressraum).
  • AAAA-Record: Auflösung in moderne IPv6-Adressen (erweiterter, zukunftssicherer Adressraum).
  • CNAME-Record: Alias-Verweis einer Domain auf eine externe Ziel-Domain.
  • TTL (Time to Live): Definiert die maximale Caching-Dauer der Records im lokalen Zwischenspeicher vor einer Neuabfrage.

3. Gateway & NAT

Die logische Schnittstelle und Verbindungskomponente zwischen dem lokalen LAN und externen Netzen (WAN).

  • Routing: Pakete an netzwerkfremde IP-Adressen werden automatisch an das Standard-Gateway weitergeleitet.
  • NAT (Network Address Translation): Das Gateway maskiert private, lokale IP-Adressen und übersetzt sie in eine einzige öffentliche WAN-IP.
